快速安装​

KubeGems Installer Operator是一个通过 Operator SDK 构建的Ansible Kubernetes控制器。通过CRD中的定义的字段对Ansible进行变量传递。Installer Operator的运行需要具备集群管理员的RBAC权限，所以在部署前需要联系您的集群管理员，以保证有足够的授权执行以下操作。

Installation​

1.执行以下命令安装KubeGems Installer控制器

  • DockerHub 源
  • 阿里云镜像源（中国大陆地区推荐）
  • GitHub 源
KUBEGEMS_VERSION="${KUBEGEMS_VERSION:-v1.20.0-beta.2}"

kubectl apply -n kubegems-installer -f \
https://github.com/kubegems/installer-operator/releases/download/${KUBEGEMS_VERSION}/centrol.yaml
KUBEGEMS_VERSION="${KUBEGEMS_VERSION:-v1.20.0-beta.2}"

wget -O- https://github.com/kubegems/installer-operator/releases/download/${KUBEGEMS_VERSION}/centrol.yaml \
| sed "s#image: kubegems#image: registry.cn-beijing.aliyuncs.com/kubegems#g"  \
| kubectl apply -f -
KUBEGEMS_VERSION="${KUBEGEMS_VERSION:-v1.20.0-beta.2}"

wget -O- https://github.com/kubegems/installer-operator/releases/download/${KUBEGEMS_VERSION}/centrol.yaml \
| sed "s#image: kubegems#image: ghcr.io/kubegems#g" \
| kubectl apply -f -

2.执行以下命令验证部署结果

kubectl get pod -n kubegems-installer

NAME                                           READY   STATUS    RESTARTS   AGE     IP                NODE              NOMINATED NODE   READINESS GATES
kubegems-installer-manager-8649bd7fcb-j286g    2/2     Running   0          15m     192.169.15.92     k8s-master3-123   <none>           <none>

Installer​

1.执行以下命令提交 installer 资源

  • DockerHub 源
  • 阿里云镜像源（中国大陆地区推荐）
  • GitHub 源
# Kubernetes 集群的容器运行时，默认是 docker，支持 containerd
CONTAINER_RUNTIME="${CONTAINER_RUNTIME:-docker}"
# 部署 KubeGems 服务使用的镜像源，默认 docker.io
REPOSITORY="${REPOSITORY:-docker.io/kubegems}"
# Kubernetes 集群中默认的 StorageClass。如没有，默认采用 local-path，所有数据将存在主机 /data 路径下
STORAGECLASS="${STORAGECLASS:-local-path}"
# Kubernetes 集群名称
CLUSTER_NAME="${CLUSTER_NAME:-kubegems-centrol}"

# 设置 Installer 环境变量，并提交开始部署
kubectl apply -f <( \
    wget -O - https://github.com/kubegems/installer-operator/releases/download/${KUBEGEMS_VERSION}/centrol.installer.template.yaml | \
        KUBEGEMS_VERSION="${KUBEGEMS_VERSION}" \
        CONTAINER_RUNTIME="${CONTAINER_RUNTIME}" \
        REPOSITORY="${REPOSITORY}" \
        STORAGECLASS="${STORAGECLASS}" \
        CLUSTER_NAME="${CLUSTER_NAME}" \
        envsubst \
)

2.观察Installer部署状态

在提交CR后，可是使用以下命令观察installer的日志输出

kubectl logs -n kubegems-installer  <installer-operator容器> -c manager -f

指导出现类似下面日志，则表示初始化完成。

您可以在创建一条ingress 规则将 kubegems dashboard 服务暴露出来，参考如下

apiVersion: extensions/v1beta1
kind: Ingress
metadata:
  annotations:
    nginx.org/proxy-buffering: "false"
    nginx.org/websocket-services: gems-dashboard
  name: gems-dashboard
  namespace: gemcloud-system
spec:
  ingressClassName: default-gateway
  rules:
  - host: console.kubegems.io
    http:
      paths:
      - backend:
          serviceName: gems-dashboard
          servicePort: 8000
        path: /
        pathType: ImplementationSpecific

此时，您需要将域名console.kubegems.io解析到 Kubernetes 集群内任意节点后，即可通过 http://console.kubegems.io:<NodePort> 访问页面。

添加管理集群​

使用 admin 身份登录 KubeGems 平台后，点击右下角 “工具” 图标，进入【平台管理】，点击 “添加集群”。

info

初次安装完成后，KubeGems 内无集群数据，需要用户自信将当前集群添加到 KubeGems 当中。

  • 将您的 kubeconfig 内容粘贴至文本框，点击“下一步”

  • 当 kubeconfig 校验成功，您可以看到如下界面后，点击“下一步”
  • 勾选 “控制集群”，并端着控制集群中的一些基本信息，如“镜像仓库”、“存储类型”等。
caution

如无法确定如何选择上述信息，保持默认即可。

  • 提交保存成功，您在“集群资源”页面即可看到新加的控制集群。

  • 点击控制集群的“详情”，即可看到集群的详细信息。

命名空间用途

observabilityKubeGems Jaeger 应用链路跟踪服务所在空间
istio-systemKubegems istio 服务治理套件所在空间
local-path-storagelocal-path-provisioner所在空间，用于创建本地文件系统的 pvc
